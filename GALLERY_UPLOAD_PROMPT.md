# Cloudflare R2 ギャラリー画像アップロード・プロンプト

このファイルを添付して、チャットで「画像をアップロードして」などの指示を送信することで、指定のローカル画像がCloudflare R2にアップロードされ、サイト上のギャラリー画面にて公開されます。

---

## 実行要件

**事前準備（ユーザー向け）:**
1. `uploads/gallery/` フォルダの中に、R2へアップロードしたい画像ファイル（.jpg, .png, .gif, .webp）を配置してください。
   ※配置済みの場合はこのステップをスキップしてOKです。
2. `.env` ファイルに `R2_ACCOUNT_ID`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`, `R2_BUCKET_NAME`, `R2_PUBLIC_DOMAIN` が正しく設定されていることを確認してください。

---

## プロンプト本文（実行内容）

AIエージェントは、以下の手順を実行してください。
**指示がない限り、JavaScriptでスクリプトの中身を書き換えることはせず、既存のツールやスクリプトをそのまま実行してください。**

### 1. アップロード処理の実行

Node.jsを使用して、用意されたアップロード自動化スクリプトを実行してください。

```bash
node scripts/upload_gallery.js
```

### 2. 実行結果の確認

実行ログを確認し、以下の項目をユーザーに報告してください：
- アップロードが成功したファイル数と画像タイトル
- 失敗したファイル数（原因・エラーの詳細）
- ギャラリーのURLへのアクセス確認の案内 (`http://localhost:3000/gallery`)

### 3. 注意点（アップロード先）
- 実際の画像ファイルは Cloudflare R2 に送信され、サーバーのストレージ容量は圧迫されません。
- `.env` で `R2_PUBLIC_DOMAIN` を指定している場合は、そのドメイン名を通じて Web 上に公開されます。
- すべての画像データ（タイトルやR2のURLなど）は MongoDB 内の `GalleryImage` コレクションに保存されます。
- 成功した画像については、ローカル（`uploads/gallery/`）から自動的に削除され、容量の節約と次回の二重アップロードを防止します。
