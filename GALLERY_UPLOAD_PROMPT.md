# Cloudflare R2 ギャラリー アップロード・同期プロンプト

このファイルを添付して「ギャラリーを追加して」と指示した場合は、通常の Copilot チャットからアップロード対象を確認し、`scripts/upload_gallery.js` の実行と処理済み画像の退避まで続けて行います。

このワークフローは**専用エージェントの起動を前提にしません**。通常の Copilot チャットで段階的に確認しながら進めてください。

---

## R2バケットのフォルダ構造について

**ローカルの `uploads/gallery/` 配下の最上位フォルダ名 = アニメ・ゲームタイトル名**として扱います。  
サブフォルダがある場合も再帰的に走査し、R2 にも同じフォルダ構造で保存されます。

```
uploads/gallery/
  女神のカフェテラス/        ← フォルダ名がシリーズ名・タグになる
    image001.png
    image002.png
  ブルーアーカイブ/
    char01.jpg
    event/
      char02.jpg
  （ルート直下の画像は「その他」に分類）
```

- 最上位フォルダ名は**必ずタグの先頭に追加**されます。
- サブフォルダ配下の画像もアップロード対象です。
- ギャラリーのトップページはフォルダ単位でシリーズカードとして表示されます（`/gallery`）。
- 各シリーズの画像一覧は `/gallery/series/<フォルダ名>` で確認できます。

---

## タイトル・タグの自動付与について

フォルダ名が作品名のため、基本的にフォルダ名がタイトル・タグになります。  
`.metadata.json` を用意すればキャラ名まで登録できます（任意）。

| 状況 | タイトル | タグ |
|---|---|---|
| R18でない画像 | フォルダ名 N | `[フォルダ名]` |
| R18の画像 | フォルダ名 R18-N | `[フォルダ名, R18]` |
| フォルダなし（ルート直下） | その他 N | `[その他]` |

---

## 実行要件

`.env` ファイルに以下が正しく設定されていることを確認してください：
- `CLOUDFLARE_API_TOKEN` — Account API Token（Workers R2 Storage: Edit 権限）
- `R2_ACCOUNT_ID`, `R2_BUCKET_NAME`, `R2_PUBLIC_DOMAIN`
- `MONGODB_URI`

---

## 処理済み画像の退避ルール

- `uploads/gallery/` は**未処理画像専用の一時置き場**として使います
- アップロードとサイト確認が終わった画像は、`uploads/gallery_archive/YYYY-MM-DD/` 配下へ移動します
- 退避先でも作品ごとのフォルダ構造は維持します
- 画像を退避した後も、`uploads/gallery/` 側の作品フォルダは削除せずに残します
- 次回のギャラリー追加前に、`uploads/gallery/` には今回アップロードしたい新規画像だけを置きます
- `uploads/gallery_archive/` 配下の画像はアップロード対象として扱いません

例:

```text
uploads/
  gallery/
    ブルーアーカイブ/
      new-image-01.png
  gallery_archive/
    2026-04-09/
      ブルーアーカイブ/
        old-image-01.png
```

## ワークフロー

### 【A】ローカル画像を R2 にアップロードしてギャラリーに反映する

新しい画像をギャラリーに追加するときの標準フローです。  
このファイルを添付して「ギャラリーを追加して」と指示された場合は、通常の Copilot チャットでアップロード対象を確認し、必要なら限定的に画像確認を行ったうえで、アップロードと退避まで続けて実行します。

#### 手順 1. 画像がフォルダに配置されているか確認する

以下のフォルダ構成になっているかを確認してください：

```
uploads/gallery/
  アニメ・ゲームタイトル名/
    image001.png
    image002.png
```

フォルダ名が作品名・シリーズ名になります。フォルダがない場合は「その他」に分類されます。
サブフォルダがある場合も、最上位フォルダ名を作品名・シリーズ名として扱います。

#### 手順 2. 画像を解析して `.metadata.json` を作成する

`uploads/gallery/` 内の**未登録画像**（既にDBに登録済みのものはスキップ）を確認し、必要な場合だけ少数ずつ画像内容を確認して、以下を判断してください：

- センシティブ（R18）な内容が含まれているかどうか

注意:
- 大量画像に対して一括で `view_image` を実行しないこと
- 画像確認が不要なら `.metadata.json` を作らず、そのまま `scripts/upload_gallery.js` を実行してよい
- チャット上の短い命令だけで専用エージェントを起動する前提にしないこと

判定ルール：

| 判定結果 | type |
|---|---|
| R18等センシティブ | `"r18"` |
| それ以外 | `"normal"` |

解析結果を `uploads/gallery/.metadata.json` として保存してください。  
キーは `uploads/gallery/` からの相対パス（例: `女神のカフェテラス/image001.png`, `ブルーアーカイブ/event/char02.jpg`）を使用します。

```json
{
  "女神のカフェテラス/00002-843823338.png": {
    "type": "normal"
  },
  "女神のカフェテラス/00023-44977716.png": {
    "type": "r18"
  }
}
```

#### 手順 3. アップロードスクリプトを実行する

```bash
node scripts/upload_gallery.js
```

- ローカルの画像を Cloudflare REST API 経由で R2 にアップロード
- MongoDB に自動登録（キャラ名タイトル＋フォルダ名タグ付き）
- `.metadata.json` はスクリプト完了後に自動削除
- **アップロード済みのローカルファイル・フォルダはそのまま残ります**

#### 手順 4. 実行結果をユーザーに報告する

- 追加された画像数・タイトル・タグ
- スキップ（既登録）件数
- エラーがあればその詳細
- サイト確認URL: `http://localhost:3000/gallery`
- このファイル添付時の標準依頼では、報告後にそのまま退避まで進めてよい

#### 手順 5. 処理済み画像を退避する

アップロード結果を確認したら、今回処理した画像を `uploads/gallery_archive/YYYY-MM-DD/` に移動してください。

このファイルを添付して「ギャラリーを追加して」と指示された場合は、この退避までを1回の依頼の既定動作とします。

推奨確認順:

1. `scripts/upload_gallery.js` の実行結果で追加件数、スキップ件数、エラー件数を確認する
2. `http://localhost:3000/gallery` で反映確認を試みる
3. サイト確認ができた場合は結果を報告し、今回処理した画像だけを `uploads/gallery_archive/YYYY-MM-DD/` へ移動する
4. サイト確認ができない場合でも、アップロード結果にエラーがなければその旨を報告したうえで、今回処理した画像だけを `uploads/gallery_archive/YYYY-MM-DD/` へ移動する
5. `uploads/gallery/` に未処理画像だけが残っており、作品フォルダ自体は空でも維持されていることを確認する

---

### 【B】R2 と MongoDB の差分を同期する

R2 に直接アップロード済みの画像を DB に登録するときに使います。

```bash
node scripts/sync_r2_gallery.js
```

- R2 の全画像を一覧取得（Cloudflare REST API 経由）
- DB 未登録の画像のみ MongoDB に追加
- フォルダ名をタイトル・タグに自動設定

---

### 【C】R2 の画像をローカルにダウンロードする

ローカルと R2 を同期したいときに使います。既存ファイルはスキップします。

```bash
node scripts/download_r2_gallery.js
```

- R2 の全画像を `uploads/gallery/` にフォルダ構造を維持してダウンロード

## スクリプト一覧

| スクリプト | 用途 |
|---|---|
| `scripts/upload_gallery.js` | ローカル→R2アップロード＋DB登録 |
| `scripts/sync_r2_gallery.js` | R2→DB差分同期 |
| `scripts/download_r2_gallery.js` | R2→ローカルダウンロード |

## 注意点
- R2 の S3 互換 API（`*.r2.cloudflarestorage.com`）は TLS 問題により使用不可。すべての操作は `api.cloudflare.com` 経由の Cloudflare REST API を使用しています。
- 画像の公開 URL は `R2_PUBLIC_DOMAIN` から生成されます（例: `https://pub-xxx.r2.dev/フォルダ名/image.png`）。
- 既に登録済みの画像（`r2Key` が一致）は自動的にスキップされます。
- すべての画像データは MongoDB の `GalleryImage` コレクションに保存されます。
- `uploads/gallery/` に処理済み画像を溜め続けると、次回の確認やチャット実行時の負荷が増えるため、反映確認後は速やかに退避してください。
- シリーズ運用を分かりやすく保つため、画像を退避しても `uploads/gallery/` 側の空フォルダは残してください。
